# 多股横向对决：2-4 只同维度结构化对比

对 $ARGUMENTS 执行 2-4 只股票的同维度横向对决，输出结构化对比矩阵与择优建议。

**对比标的**：同一行业/赛道的 2-4 家上市公司，或用户明确指定的多只可比标的。一次只比 2-4 只，不做全市场扫描（全市场筛选用 `/industry-funnel`）。

## 适用场景

当用户说"茅台 vs 五粮液""腾讯 vs 阿里 vs 美团""中国啤酒三巨头对比"时：
1. 先用本 Skill 把财务与估值结构化维度对齐，输出对比矩阵。
2. 在矩阵基础上补护城河/管理层/确定性等定性维度裁决。
3. 输出择优建议与关键分歧点，让读者自己权衡。

与 `/industry-funnel` 的区别：漏斗从全市场逐层精选，本 Skill 只做"已知候选"的横向对决。

## 设计原则

- **工具只覆盖财务与估值结构化维度**：护城河深度、管理层诚信、长期确定性等定性维度由分析者在此基础上裁决，不由工具凭空打分。
- **结构化领先 ≠ 投资结论**：净利润大、估值低只是必要条件，不是充分条件；便宜背后往往有原因（治理塌方、渠道倒挂、周期顶部），必须呈现反面。
- **对齐口径**：所有数值对齐到同一报告期（最新公共年报期），避免跨期比对的统计幻觉（如五粮液 2025Q1 因会计调整压低基数导致高增速）。

## 执行流程

### 第一步：确认对比标的与口径

把 `$ARGUMENTS` 解析为 2-4 个明确代码或公司名。若用户给的是公司名，先用 WebSearch / 妙想确认 A 股/港股代码；跨市场（A 股 vs 港股 vs 美股）对比时必须显式标注各自币种，汇率口径统一为人民币（按当日中间价）。

**约束**：少于 2 只或多于 4 只时，请用户调整后再执行。

### 第二步：拉取结构化对比数据

复用 `tools/lxr_data.py compare` 子命令（理杏仁首选，自动降级妙想/免费源），一站取齐每只股票的财务与估值并自动对齐：

```bash
# 默认输出对比矩阵 Markdown（含★领先标记、越低/越高越好方向、平局提示、_source 来源标注）
python tools/lxr_data.py compare {code1} {code2} [{code3}] [{code4}]

# 需要机器可读 payload 供后续加工时（--json 输出到 stdout，可自行重定向）
python tools/lxr_data.py compare {code1} {code2} --json
```

`_source` 标注遵循 `docs/channel-capability-matrix.md`：理杏仁拉到标 `lixinger`，妙想降级补数标 `lixinger+mx-data`，免费源兜底标 `multi`。报告里每张表的来源/ref 列必须与此一致，不得虚构来源。

工具已对齐的维度（共 8 财务 + 5 估值）：

| 类别 | 维度 | better 方向 |
|------|------|-----------|
| 财务 | 营业收入 / 归母净利润 / 净利率 / 毛利率 / ROE / 经营现金流 / 货币资金 / EPS | 越高越好 |
| 估值 | PE(TTM) / PB / PE分位(10y) / PB分位(10y) / 股息率 | PE/PB/分位越低越好；股息率越高越好 |

净利率/毛利率/ROE 由原始指标派生（比值）；全空维度不进矩阵；元单位在表中换算为亿。

### 第三步：补定性裁决维度

工具矩阵只回答"谁在财务与估值上领先"，不回答"谁更值得投"。在矩阵之上补三组定性维度，每维都需呈现正反两面：

1. **护城河差异**：定价权、网络效应、转换成本、规模效应、监管壁垒——逐项指出谁更宽更持久，**但另一方面**列出领先者护城河可能被颠覆的路径。
2. **管理层与治理**：诚信度、资本配置历史、股东回报政策、关联交易——指出谁治理更优，**但另一方面**说明劣势方可能正在修正或领先方有未暴露的治理风险。
3. **长期确定性**：10 年后这家公司还在吗？被什么颠覆？确定性差异往往比当期估值更重要。

### 第四步：输出择优建议与关键分歧

输出以下结构（落到 `reports/多公司对比-{主标的}-{YYYYMMDD}.md` 或 `reports/{主标的}vs{次标的}-对比-{YYYYMMDD}.md`，命名风格与现有 `茅台vs五粮液-护城河与估值深度对比` 一致）：

1. **一句话结论**：谁在结构化维度领先 + 便宜背后是否有原因（80 字内）。
2. **对比总表**：核心维度行 × 股票列，附★领先标记（工具矩阵直出或重排）。
3. **财务对决**：营收/净利润/毛利率/净利率/ROE/现金流关键项，跨期对比时标注会计调整或基数效应。
4. **估值对决**：PE/PB/分位/股息率，便宜的一方必须附带"便宜的原因"或"便宜的代价"。
5. **护城河/管理层/确定性**：定性裁决，每项含反面论据。
6. **择优建议**：结构化领先股 + 定性裁决后的最终建议分层（激进/稳健/保守 → 是否介入、介入区间），明确"领先≠买入"。
7. **关键分歧点**：列出两方分歧最大的 2-3 个维度（如估值便宜 vs 治理塌方），让读者自己权衡。
8. **数据来源**：每张表的来源/ref 列，`_source` 标注与第二步一致。

## 重要注意事项

1. **结构化领先只是起点**——茅台净利润是五粮液 9 倍、ROE 4 倍，但投资结论仍要看护城河宽度与确定性，不是单纯比谁的指标漂亮。
2. **便宜背后的代价**——纸面估值更低的一方往往有治理塌方、渠道倒挂、周期顶部等隐藏成本，必须逐项呈现，不能只报便宜。
3. **跨期对比防统计幻觉**——增速对比要剔除会计调整/基数效应（见五粮液 2025Q1 案），否则会把统计幻觉误读为经营改善。
4. **跨市场币种统一**——A 股 vs 港股 vs 美股对比必须统一币种，防止市值/营收量级误读。
5. **正反两面**——每个核心判断（如"茅台确定性更高"）必须附带反面论据（如"国企治理 7 年 5 换帅"），让读者权衡。
6. **诚实说不确定**——当定性维度（如 10 年后是否被颠覆）无法判断时，明确标注"数据不足"或"不确定"，不要用推测填满框架伪装确定性。
7. **数据来源标注**——所有关键数据至少 2 个来源交叉验证；估计值必须注明"估计"、口径与假设。
